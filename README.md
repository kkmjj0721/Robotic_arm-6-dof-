# `tool_modeled_link` RCM 运动学仿真说明

## 1. 项目目的

实现了一个基于 MuJoCo 的纯运动学仿真，用来演示 `tool_modeled_link` 围绕空间中一个固定点做：

- `pitch` 俯仰
- `yaw` 偏航
- `insertion` 插入/回退

固定点在每次启动仿真或按下重置键时，在当前可达工作空间内随机采样一个可行点，然后整个回合都围绕这个点做 RCM 运动

## 2. 工程结构
- `config.py`
  - 全局配置
  - 默认模型路径
  - 控制步长、采样边界、阻尼和误差阈值
- `kinematics.py`
  - MuJoCo 名称索引解析
  - 关节配置写入
  - `tool_modeled_link` 轴线/端点重建
  - 几何观测量计算
- `controller.py`
  - 键盘命令映射
  - 目标状态更新
  - 基于阻尼最小二乘的 5 自由度运动学求解
- `sim.py`
  - 随机场景采样
  - 单步安全检查
  - MuJoCo + GLFW 可视化循环
- `main.py`
  - 启动入口
- `model/Simplify_model/`
  - MuJoCo XML、URDF、Xacro 模型文件

## 3. 运行环境

当前代码依赖：

- Python 3.10+
- `mujoco`
- `numpy`
- `glfw`

## 4. 如何启动

进入到sim/simplify_sim中，直接运行：

```bash
python3 main.py
```

## 5. 键盘操作

窗口启动后，按键如下：

- `I`：俯仰上抬
- `K`：俯仰下压
- `J`：偏航左转
- `L`：偏航右转
- `U`：插入更深
- `O`：向外回退
- `N`：重新随机生成一个固定点和初始场景
- `X`：退出程序

这些按键的含义是“修改目标”，不是直接改关节值。

## 8. 技术实现原理

### 8.1 基本建模对象

当前被约束的不是主工具轴，也不是 `tool_tip`，而是：

- `tool_modeled_link`

这是因为当前模型中新增了一个单独的末端建模杆件，它：

- 挂接在 `tool_inner_link` 末端
- 具有固定 `30 deg` 俯仰偏置
- 连接点位于自身长度的 `1/4` 位置

所以当前仿真是直接以 `tool_modeled_link` 作为 RCM 受控杆。

### 8.2 随机固定点如何生成

程序启动时不是随便在空间里取一个点，而是先随机采样一组安全姿态，再从这组姿态的当前杆轴上取一点作为固定点。

采样条件包括：

- 主动关节要远离关节边界
- 最小奇异值不能太小
- 杆的近端和远端不能太低
- 固定点不能落在杆端点附近

这样做的目的是：

- 保证固定点一定是当前模型可达的
- 避免一开始就在工作空间边缘
- 避免一开始就接近奇异姿态

### 8.3 纯运动学而不是动力学

当前程序是纯运动学实现，不做动力学积分。

代码里采用的是：

- 直接写入 `data.qpos`
- 清零 `data.qvel`
- 清零 `data.qacc`
- 调用 `mujoco.mj_forward`

没有用到：

- `mujoco.mj_step`
- 力/力矩控制
- 执行器 `ctrl`
- 接触动力学驱动的运动推进

所以这个工程本质上是在做：

- 给定关节配置
- 前向计算几何状态
- 根据几何误差反解下一组关节配置

### 8.4 杆的几何信息怎么重建

仿真里不是直接把 `body` 原点当作杆端点，因为 `tool_modeled_link` 的 body 原点不在杆端。

程序会通过：

- body 的世界位置 `xpos`
- body 的旋转矩阵 `xmat`
- geom 的局部中心偏移 `geom_pos`
- geom 的半长度 `geom_size[1]`
- tip site 的世界位置

重建出：

- 杆近端点 `rod_prox_world`
- 杆远端点 `rod_tip_world`
- 杆方向 `rod_dir`
- 杆长度 `rod_length`

这里默认 `tool_modeled_link` 的轴方向是 body 局部 `-Y` 轴。

### 8.5 固定点误差怎么定义

RCM 误差不是某两个点的距离，而是：

- 固定点到杆轴线的最短距离

具体做法是：

1. 先计算固定点相对于杆近端点的向量 `delta`
2. 计算 `delta` 在杆方向上的投影
3. 用 `delta - 投影` 得到垂直于杆轴的分量
4. 这个垂直分量的长度就是 `pivot_error_norm`

这比直接看某个 link 原点到固定点的距离更符合 RCM 本质。

### 8.6 插入量怎么定义

插入量不是独立关节，而是几何量：

- 固定点沿杆方向距离杆近端点有多远

也就是：

- `insertion = dot(pivot_world - rod_prox_world, rod_dir)`

所以当前代码里的“插入”是沿当前受控杆方向前后移动的任务量。

### 8.7 控制器如何工作

用户按键后，不是直接改关节，而是先更新目标：

- 目标方向由 `yaw_ref + pitch_ref` 决定
- 目标插入量由 `insertion_ref` 决定

然后控制器根据固定点和目标方向，反推出：

- 期望近端点 `desired_prox`
- 期望远端点 `desired_tip`

接着构造误差：

- 当前近端点和目标近端点的差
- 当前远端点和目标远端点的差

最后用阻尼最小二乘求一组关节增量 `dq`。

### 8.8 为什么同时约束近端点和远端点

如果只约束杆方向，插入量可能漂。
如果只约束某一个端点，杆方向可能不稳。

当前写法同时约束：

- 近端点
- 远端点

相当于直接约束整根杆的位姿，而不是只约束一部分几何量。

这是当前精简版控制器最核心的设计。

### 8.9 为什么要加阻尼和回中项

控制器里还有两个稳定性设计：

1. 阻尼项
   - 在接近奇异时增大阻尼
   - 避免矩阵求解发散

2. 关节回中偏置
   - 鼓励解不要长期贴着关节边界跑
   - 提升数值稳定性和可持续运动能力

### 8.10 为什么还要做单步接受检查

每次求出 `q_next` 后，程序不会无条件接受，还会检查：

- `pivot_error_norm` 是否超过阈值
- 误差跳变量是否过大
- `sigma_min` 是否跌破硬阈值

如果不满足，就拒绝这一步，保持上一状态。

这样做的目的是：

- 防止一次数值更新把姿态推坏
- 防止突然进入奇异区域

## 9. 当前版本的限制

- 当前是数值运动学求解，不是解析闭式解
- 当前是仿真版，不是硬件控制版
- 仿真内部允许一起求解 `tool_yaw / tool_pitch`，真实机械臂上的这两个万向节是被动，硬件实现需要换思路
